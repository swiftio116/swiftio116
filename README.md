## Code Quality

I try to keep iOS projects simple, readable and testable.

- ViewController: UI, user actions and state rendering.
- ViewModel / Presenter: presentation logic.
- Service layer: networking, Firebase and external dependencies.
- Coordinator: navigation and module assembly.
- Protocols: abstractions for testability.
- Mocks: unit tests without real network or Firebase.
- `.gitignore`: clean repository without private configs and service files.
